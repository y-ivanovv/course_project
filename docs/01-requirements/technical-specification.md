# Техническое задание

**Система:** «Электронная библиотека» (Library)
**Траектория:** Б — Веб-ориентированная
**Автор:** Иванов Юрий Сергеевич, гр. ПИЖ-б-о-23-1

## 1. Общие сведения
- **Наименование системы:** информационная система «Электронная библиотека».
- **Назначение:** автоматизация ведения каталога книг, поиска изданий и учёта выдачи/возврата
  с разграничением ролей пользователей.
- **Заказчик (учебный):** межинститутская базовая кафедра, дисциплина «Программная инженерия».

## 2. Основания и цель разработки
- **Основание:** задание на курсовой проект по дисциплине «Программная инженерия».
- **Цель:** сократить время обслуживания читателей и обеспечить достоверный учёт фонда и выдач.

## 3. Требования к системе

### 3.1. Требования к функциональным характеристикам
Перечень подсистем/модулей:
- **Модуль пользователей:** регистрация, вход/выход, роли (USER/LIBRARIAN), создание библиотекарей.
- **Модуль каталога:** добавление и удаление книг (по ISBN), просмотр с пагинацией.
- **Модуль поиска:** полнотекстовый поиск по названию, описанию, автору, жанру и ISBN.
- **Модуль выдачи:** выдача и возврат книг с привязкой к пользователю.

Функциональные требования (см. также Таблицу 1.2 ПЗ):

| Код | Требование |
|-----|------------|
| FR-1 | Регистрация и аутентификация пользователя (сессия) |
| FR-2 | Просмотр каталога с постраничной навигацией (limit/offset) |
| FR-3 | Полнотекстовый поиск по всем полям книги |
| FR-4 | Просмотр карточки книги |
| FR-5 | Выдача книги (создание записи о выдаче) |
| FR-6 | Возврат книги (только пользователем, оформившим выдачу) |
| FR-7 | Добавление книги (роль LIBRARIAN) |
| FR-8 | Удаление книги по ISBN (роль LIBRARIAN) |
| FR-9 | Создание учётной записи библиотекаря (роль LIBRARIAN) |

### 3.2. Требования к архитектуре (PCMEF)
- Слоистая архитектура Presentation · Control · Mediator · Entity · Foundation.
- Зависимости направлены строго сверху вниз; взаимодействие слоёв — через интерфейсы
  сервисов (`BookService`, `UserService`, `LoanService`) и репозиториев Spring Data.
- Подробно — в [docs/02-architecture](../02-architecture/architecture.md).

### 3.3. Требования к надёжности
- Транзакционность операций выдачи/возврата (`@Transactional`).
- При сбое обновления статуса книги (Elasticsearch) запись о выдаче (PostgreSQL) откатывается.
- Сохранность данных пользователей и выдач в реляционной БД.

### 3.4. Требования к безопасности
- Хранение паролей только в виде BCrypt-хеша.
- Разграничение доступа по ролям (перехватчик `AuthInterceptor` + `@RequireRole`).
- Сокрытие чувствительных полей в ответах API (DTO `UserResponse`).
- Валидация входных данных, единый формат ошибок (`ErrorResponse`).

### 3.5. Требования к интерфейсу
- Веб-интерфейс не менее 5 страниц (вход, регистрация, каталог/поиск, карточка книги,
  панель библиотекаря), навигация и валидация форм на клиенте.

### 3.6. Требования к данным
- Реляционная модель (PostgreSQL): таблицы `users`, `loans`, нормализация до 3НФ.
- Каталог книг — индекс `books` в Elasticsearch (русский анализатор).
- DDL — в [docs/03-database/schema.sql](../03-database/schema.sql).

## 4. Требования к составу и параметрам технических средств
- Сервер: JRE 21+, доступ к PostgreSQL 16 и Elasticsearch 8.11.
- Клиент: современный веб-браузер.
- Среда разработки: JDK 21, Node.js 18+, Maven 3.8+, Docker 20+.
