# ADR-003: Учёт выдач в отдельной таблице PostgreSQL

- **Статус:** Принято
- **Дата:** 2026-05

## Контекст
Изначально выдача/возврат только переключали статус книги (`AVAILABLE`/`BORROWED`) в Elasticsearch
и никак не фиксировали, **кто** взял книгу. Следствие — вернуть книгу мог любой пользователь, а не
тот, кто её взял.

## Рассмотренные варианты
1. **Поле `borrowerId` в документе книги (ES)** — смешивает каталог и операционные данные, нет истории.
2. **Отдельная таблица `loans` в PostgreSQL** — нормальная реляционная модель, история выдач,
   транзакции.

## Решение
Добавлена сущность/таблица **`Loan`** (`book_id`, `user_id`, `borrowed_at`, `returned_at`).
Активная выдача — запись с `returned_at IS NULL`. Логика вынесена в `LoanService`:
- выдача создаёт `Loan` и переводит книгу в `BORROWED`;
- возврат разрешён только пользователю, оформившему выдачу (иначе 403), затем книга → `AVAILABLE`.

## Последствия
- (+) Достоверный учёт «кто держит книгу», история выдач, корректная защита возврата.
- (+) Разделение каталога (ES) и операционных данных (PostgreSQL).
- (−) Операция затрагивает два хранилища; целостность обеспечивается в рамках сервисного метода
  (`@Transactional` для PostgreSQL; сбой обновления ES откатывает запись о выдаче).
