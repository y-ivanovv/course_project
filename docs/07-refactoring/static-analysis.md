# Отчёт статического анализа кода

Статический анализ выполнен для обеих частей проекта: серверной (Checkstyle) и клиентской (ESLint).

## 1. Backend — Checkstyle

- Инструмент: Checkstyle 9.3 (плагин `maven-checkstyle-plugin`), профиль проверок — `sun_checks`.
- Запуск: `mvn checkstyle:checkstyle` (полный машинный отчёт — [checkstyle-result.xml](checkstyle-result.xml)).
- Проанализировано файлов: 33.

| Категория замечания | Кол-во | Характер |
|---------------------|-------:|----------|
| FinalParameters | 151 | Стилистика (рекомендация `final` у параметров) |
| DesignForExtension | 111 | Стилистика (Sun-конвенция расширяемости) |
| LineLength | 90 | Форматирование (длина строки) |
| HiddenField | 71 | Затенение поля параметром конструктора/сеттера |
| JavadocVariable | 69 | Отсутствие Javadoc у полей |
| LeftCurly | 69 | Расположение фигурной скобки |
| MissingJavadocMethod / JavadocMethod | 89 | Отсутствие Javadoc у методов |
| **Всего** | **698** | — |

**Вывод.** Все замечания относятся к стилистическим конвенциям профиля `sun_checks`
(требование `final` у параметров, обязательный Javadoc для каждого поля/метода, длина
строки, оформление скобок). Дефектов уровня ошибок (потенциальные NPE, мёртвый код,
нарушение контрактов) Checkstyle не выявил. Стилистические замечания не влияют на
работоспособность и устраняются постепенно; критические для читаемости (длина строк,
расположение скобок) учтены при рефакторинге.

## 2. Frontend — ESLint

- Инструмент: ESLint (flat config), плагины `react-hooks`, `react-refresh`.
- Запуск: `npm run lint`.
- Результат: **0 ошибок, 0 предупреждений**.

В ходе рефакторинга устранены: неиспользуемые импорты `React`, неиспользуемые переменные,
нарушения правил хуков (`react-hooks/set-state-in-effect`, корректные зависимости эффектов).

## 3. Итог

Статический анализ покрывает весь стек. Клиентский код проходит линтер без замечаний;
серверный код проверен Checkstyle — выявлены только стилистические рекомендации, дефектов
не обнаружено. Как направление развития зафиксировано подключение SonarQube для сводной
оценки качества и технического долга.
